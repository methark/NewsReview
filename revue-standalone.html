The News Revue — no-backend edition

The Trinary Method: a story is only shown here once at least 3 independent, high-fidelity sources confirm it. Biased, speculative, and unattributed-opinion language is filtered out before you ever see it.

This version runs entirely in your browser — no server of any kind. Browsers can't fetch RSS feeds cross-origin without permission, so this reads them through public CORS-proxy services we don't control; if a feed shows as unreachable, that proxy layer (not your setup) is usually why. It cannot search open search engines directly (Google/Bing block exactly this, by design) — "Search the Web" here searches this run's fetched articles, the same 30-40 curated outlets as always.

Fetching sources through the CORS proxy and running the Trinary Method…

Runs entirely client-side: no PHP, no server of any kind. Fetches the same ~40 curated World/Science/Finance RSS feeds as the PHP-backed edition, routed through a public CORS-proxy service (with a fallback proxy if the first fails) since browsers can't fetch cross-origin feeds directly. This is inherently less reliable than a real backend — proxy downtime, rate limits, or policy changes are outside this page's control.

Results are cached in this browser (localStorage) for an hour so repeated searches or category changes don't re-fetch every feed. "Start auto-refresh" polls again every few minutes until you click "Stop" — deliberately not a tight non-stop loop, since hammering a free proxy as fast as possible gets it to block this page within seconds.
